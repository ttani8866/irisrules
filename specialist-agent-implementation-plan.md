# 業務改善ツールからAIエージェントへ — 専門エージェント実装計画

**対象期間:** 2026年3月〜8月  
**目標:** 8月末までに各部署へ専門エージェントを配置

## 設計思想

業務改善ツールは最終目的ではない。各ツールは8月末に配置する「専門エージェント」の一部機能として設計する。

**エージェント化の3原則:**
1. **全入出力をデータとして保存** - 学習ループの原材料
2. **ツール間でデータ共有** - 格納場所と形式を統一
3. **人間判断ポイントを明示** - AI/人間の境界線が設計図になる

---

## 6つの専門エージェント

### Agent 1: デジタル広告エージェント
**配置先:** デジタル部門  
**優先度:** 最優先

| 要望No. | 機能 | 役割 |
|---------|------|------|
| No.3 | レポート数値Wチェック・アラート | エラーハンドリング |
| No.1 | WEB広告データ統合・自動インサイト | 入力データ源・分析 |
| No.2 | 検索広告TD自動生成・改善提案 | 企画出力 |
| No.4 | Webレポート自動テンプレート化 | 報告書出力 |

**着手理由:**
- 誤入稿の実損リスク遮断（投資対効果が最も見えやすい）
- データ構造化済み（BigQuery、GA）で開発リスク低
- 現場の信頼獲得が最速

---

### Agent 2: メディアプランニングエージェント
**配置先:** メディア部門  
**優先度:** 最優先（Agent 1と並行着手）

| 要望No. | 機能 | 役割 |
|---------|------|------|
| No.6 | スポット案分析・改善提案 | 中核分析機能 |
| No.7 | 広告効果算出・媒体資料分析 | データ入力源 |
| No.9 | 進行表自動生成・CM枠自動割付 | プラン出力 |
| No.8 | 全国媒体社への見積打診・メール管理 | 配信機能（最後に実装） |

**着手理由:**
- 広告会社の提案力に直結
- No.6→No.7→No.9の学習ループが最大の燃料
- 蓄積データが競合に追いつけない資産になる

**留意:** No.8（メール送信系）は人間レビュー接合点。慎重に設計し、最後に実装。

## 実行スケジュール

### Phase 1: 基盤設計＋先行アプリ着手（3月〜4月）
**やること:**
- 全ツール共通のデータ基盤設計（格納場所・形式・命名規則統一）
- 人間判断ポイントの設計書作成
- Agent 1のNo.3（Wチェック・アラート）を最速開発・投入
- Agent 2のNo.6（スポット案分析）の要件定義・開発着手

**ゴール:**
- データ基盤設計確定
- No.3が現場で稼働、「AIツールは使える」という信頼獲得
- No.6の開発開始

---

### Phase 2: 主要ツール開発＋初期エージェント統合（5月〜6月）
**やること:**
- Agent 1の残り機能（No.1、No.2、No.4）開発
- Agent 2の残り機能（No.7、No.9）開発
- Agent 3（No.10、No.11）の開発着手
- **Agent 6（経営意思決定）の優先機能着手:**
  - 経営ダッシュボード
  - 売上・利益予測
  - 経営会議資料自動生成
- No.6→No.9のデータ連携実証（エージェント統合の原型）

**ゴール:**
- Agent 1とAgent 2の主要機能がツールとして稼働
- 現場にAIツール使用習慣が定着
- ツール間データ連携が実証済み
- **経営層が経営ダッシュボードを日常利用開始**

---

### Phase 3: エージェント統合＋全部署配置（7月〜8月）
**やること:**
- Agent 1、Agent 2に統合エージェント機能実装（ツール間自動連携）
- Agent 3の残り機能（No.17、No.18、受注確率予測、フォローアップ自動化）追加
- Agent 4、Agent 5の優先機能開発・配置
- **Agent 6の残り機能追加:**
  - キャッシュフロー予測
  - リスク分析・早期警戒システム
  - 事業ポートフォリオ分析
- 全6エージェントの部署配置と運用開始

**ゴール:**
- 各部署に専門エージェント配置
- Agent 1とAgent 2は「人間は確認・承認のみ」の状態に接近
- Agent 3〜5はツール段階で稼働、データ蓄積開始
- **経営層が予測データとリスクアラートに基づいた意思決定を実施**

## 留意事項

1. **全項目の完成を目指さない** - エージェントの「骨格」が動いていれば、個別機能は9月以降追加可能
2. **データ基盤設計が最優先** - これを怠ると6つのエージェントが「6つの別々のツール」になる
3. **No.8（メール送信系）は慎重に** - 人間レビュー接合点。送信より戻ってくる見積もりデータの構造化・比較分析に重心
4. **外販 or 内製を先に決める** - Agent 2のNo.6とNo.9はMeisisネットワーク向けパッケージとして外販可能性あり
5. **データガバナンス確立** - プライベートクラウド上のクローズドLLM環境設計をプロジェクト開始前に確立
6. **グループ連携への布石** - VAZデータ統合は9月以降だが、接続ポイントを設計段階で確保

## 判断基準

**すべての設計判断に適用する基準:**

> 「この機能が生み出すデータは、学習ループに入るか？」

学習ループに入らない機能 = 時間とともに価値減衰する消耗品  
学習ループに入る機能 = 時間とともに精度が上がり、競合が追いつけない資産

Agent 2のスポット案分析（No.6）と枠割付（No.9）の精度が学習ループで積み上がる部分に、広告会社としての競争優位が生まれる。
